# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | ✅        |

## Reporting a Vulnerability

If you discover a security vulnerability in Headcount Zero, please report it responsibly.

**Do not open a public issue.**

Instead, please use [GitHub's private vulnerability reporting](https://github.com/DUBSOpenHub/headcount-zero/security/advisories/new) or contact the maintainers directly.

### What to include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

### Response timeline

- **Acknowledgment:** Within 48 hours
- **Assessment:** Within 1 week
- **Fix:** As soon as reasonably possible

## Security Considerations

Headcount Zero generates startup planning documents using AI. Keep in mind:

- **Don't include real credentials or secrets** in your startup idea prompts
- **Review generated output** before sharing — AI can produce inaccurate information
- **Generated legal checklists are not legal advice** — always consult a real attorney
- **Financial models are estimates** — validate assumptions before making real decisions
